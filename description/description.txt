Проект - имитатор ЭКГ сигнала включает в себя настройку микроконтроллера(МК) и трех периферийных устройств, таких как АЦП, дисплей и динамик.
        Функциональное назначение:
  Имитация ЭКГ сигнала пациента в 4 состояниях.
  Отображение кривой ЭКГ на пиксельном TFT дисплее.
  Обработка данных с АЦП для переключения между имитируемыми состояниями.
  Звуковая индикация для имитируемых состояний.
  Ограничения: программа имитирует 4 состояния пациента, соответственно, имеет ограничения по точной настройке ЧСС.

        Алгоритм программы: 
  Инициализация периферийных модулей (SPI, ADC, TIM)
  Очистка дисплея
  Запуск бесконечного цикла имитации состояний пациента:
  Конвертация аналогового сигнала через АЦП
  Согласно полученному значению АЦП установка одного из состояний имитации пациента
  Отображение дисплеем визуальной информации
  Звуковое сопровождение сердечных сокращений

        Краткое описание алгоритма программы:
  Функции SystemClock_Config() и SysTick_Handler() отвечают за настройку 
  системной частоты и за инкрементацию счетчика HAL, необходимой для точного 
  выставления задержки в ходе работы программы.
  Функции TFT_init() и TFT_clear() отвечают за настройку работы дисплея в 
  портретной ориентации с указанием на структуру используемого модуля SPI для 
  получения информации дисплеем и его очистку.
  В бесконечном цикле while() главной функции программы main() используются 
  условные операторы if() для проверки полученного значения ADC с целью установки 
  необходимого имитируемого состояния пациента. Запускается цикл for() отрисовки 
  имитируемого ЭКГ сигнала с помощью функции TFT_drawLine() и звуковой 
  индикации с помощью функции pwmSetPulse(). В ходе работы цикла по отрисовке 
  сигнала и звукового сопровождения на каждой итерации цикла отрисовки идет 
  проверка условия соответствия полученного значения ADC для дальнейшего 
  выполнения цикла для текущего имитируемого состояния пациента. Если значение 
  ADC перестает попадать в нужный диапазон, программа очищает дисплей и 
  останавливает звуковое сопровождение переходя к другому режиму имитации 
  пациента.

        Используемые методы: 
  Для реализации программы были написано 3 библиотеки и 1 взята извне:
        spi.c
  Библиотека, содержащая инициализацию и настройку работы модуля SPI для 
  передачи графической информации ЭКГ сигнала на дисплей. Метод spi2() отвечает за 
  инициализацию пинов для работы SPI и самого модуля SPI. Метод pinsInit() отвечает 
  за инициализацию пинов RS, CS и RESET, необходимых для работы дисплея.
        tim.c
  Библиотека, содержащая инициализацию и настройку работы модуля TIM в 
  режиме широтно-импульсной модуляции (ШИМ) для звуковой индикации 
  синхоронно с отображением графической информации ЭКГ сигнала. Метод tim4() 
  отвечает за инициализацию пинов для работы TIM и самого модуля TIM. Метод 
  pwmSetPulse(uint16_t pulse) отвечает за инициализацию и работу таймера в режиме 
  ШИМ, также метод позволяет извемять скважность ШИМ с помощью передаваемого 
  в метод значения pulse.
        adc.c
  Библиотека, содержащая инициализацию и настройку работы модуля ADC. 
  Метод adc() отвечает за инициализацию модуля ADC для работы в непрерывном 
  режиме и настройку канала ADC с высокой скоростью работы. Метод 
  HAL_ADC_ConvCpltCallback() отвечает за действия по окончании конвертации 
  модуля ADC, а именно за получение нового значения ADC.
        display.c (взята извне)
  Библиотека, содержащая инициализацию и настройку дисплея, а также 
  различные методы для отрисовки фигур на дисплее, его очистки, отправки данных 
  по интерфейсу SPI.
